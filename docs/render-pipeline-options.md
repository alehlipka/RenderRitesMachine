# Варианты рендер пайплайна

Документ описывает различные варианты рендер пайплайнов, которые можно реализовать в RenderRites Machine на базе OpenGL 4.6.

## Текущий пайплайн

**Forward Rendering (Прямой рендеринг)**
- Прямой рендеринг всех объектов в один проход
- Рендеринг каждого объекта отдельно
- Stencil buffer для идентификации объектов
- Outline rendering как отдельный проход

**Преимущества:**
- Простота реализации
- Низкие требования к памяти
- Хорошо работает для небольшого количества источников света

**Недостатки:**
- Проблемы с производительностью при большом количестве источников света
- Сложность реализации сложных эффектов освещения

---

## Вариант 1: Forward+ (Tiled Forward Rendering)

**Описание:** Улучшенная версия forward rendering с разбиением экрана на тайлы для эффективной обработки множества источников света.

### Архитектура:
1. **Geometry Pass** - Рендеринг геометрии в G-Buffer (опционально, можно использовать только depth)
2. **Light Culling Pass** - Разбиение экрана на тайлы (16x16 или 32x32 пикселя), определение видимых источников света для каждого тайла
3. **Shading Pass** - Рендеринг с учетом только видимых источников света

### Преимущества:
- ✅ Поддержка множества источников света (100+)
- ✅ Относительно простая реализация
- ✅ Хорошая производительность на современных GPU
- ✅ Совместим с текущей архитектурой

### Недостатки:
- ⚠️ Требует compute shaders (OpenGL 4.3+)
- ⚠️ Сложнее отладка

### Реализация:
- Compute shader для light culling
- Uniform buffer objects (UBO) для передачи данных о светах
- Tiled light lists в shared memory

### Сложность: Средняя
### Производительность: Высокая
### Рекомендуется для: Игр с множеством динамических источников света

---

## Вариант 2: Deferred Rendering

**Описание:** Классический deferred пайплайн с разделением на geometry pass и lighting pass.

### Архитектура:
1. **Geometry Pass** - Рендеринг всех объектов в G-Buffer:
   - Position (RGB) + Depth (A)
   - Normal (RGB) + Roughness (A)
   - Albedo (RGB) + Metallic (A)
   - Optional: Emissive, AO
2. **Lighting Pass** - Вычисление освещения на основе G-Buffer
3. **Post-Processing Pass** - Применение эффектов (bloom, tone mapping, etc.)

### Преимущества:
- ✅ Отличная производительность при множестве источников света
- ✅ Легко добавлять новые типы источников света
- ✅ Идеально для PBR (Physically Based Rendering)
- ✅ Хорошая база для расширенных эффектов (SSAO, SSGI)

### Недостатки:
- ⚠️ Высокие требования к памяти (несколько render targets)
- ⚠️ Проблемы с прозрачностью (требует forward pass для прозрачных объектов)
- ⚠️ Не поддерживает MSAA напрямую (требует дополнительные техники)

### Реализация:
- Multiple Render Targets (MRT) для G-Buffer
- Fullscreen quad для lighting pass
- Forward pass для прозрачных объектов

### Сложность: Высокая
### Производительность: Очень высокая (для множества источников света)
### Рекомендуется для: Больших сцен с множеством источников света, PBR рендеринга

---

## Вариант 3: Hybrid Deferred-Forward

**Описание:** Комбинация deferred и forward рендеринга для оптимального баланса.

### Архитектура:
1. **Deferred Pass** - Рендеринг непрозрачных объектов в G-Buffer
2. **Lighting Pass** - Вычисление освещения для непрозрачных объектов
3. **Forward Pass** - Рендеринг прозрачных объектов с forward rendering
4. **Post-Processing** - Применение эффектов

### Преимущества:
- ✅ Лучшее из обоих миров
- ✅ Поддержка прозрачности
- ✅ Эффективная обработка множества источников света для непрозрачных объектов

### Недостатки:
- ⚠️ Сложность реализации
- ⚠️ Два разных пайплайна для управления

### Сложность: Очень высокая
### Производительность: Очень высокая
### Рекомендуется для: Профессиональных игровых движков

---

## Вариант 4: Clustered Forward Rendering

**Описание:** Продвинутая версия Forward+ с 3D кластеризацией источников света.

### Архитектура:
1. **Geometry Pass** - Рендеринг геометрии
2. **Clustering Pass** - Разбиение пространства камеры на 3D кластеры (например, 16x9x24)
3. **Light Assignment** - Привязка источников света к кластерам
4. **Shading Pass** - Рендеринг с учетом кластеров

### Преимущества:
- ✅ Поддержка очень большого количества источников света (1000+)
- ✅ Эффективная работа с точечными и направленными источниками света
- ✅ Хорошая производительность

### Недостатки:
- ⚠️ Сложная реализация
- ⚠️ Требует compute shaders
- ⚠️ Сложнее отладка

### Сложность: Очень высокая
### Производительность: Очень высокая
### Рекомендуется для: AAA игр с экстремальным количеством источников света

## Вариант 5: Улучшенный Forward с Shadow Mapping

**Описание:** Текущий пайплайн с добавлением shadow mapping и улучшенным освещением.

### Архитектура:
1. **Shadow Pass** - Рендеринг теней (directional, point, spot lights)
2. **Main Pass** - Рендеринг с учетом теней
3. **Post-Processing** - Базовые эффекты

### Преимущества:
- ✅ Минимальные изменения в текущей архитектуре
- ✅ Реалистичные тени
- ✅ Хорошая производительность для небольшого количества источников света

### Недостатки:
- ⚠️ Ограничения по количеству источников света
- ⚠️ Проблемы с качеством теней (aliasing)

### Дополнительные улучшения:
- Cascaded Shadow Maps (CSM) для directional lights
- Percentage-Closer Filtering (PCF) для сглаживания теней
- Variance Shadow Maps (VSM) или Exponential Shadow Maps (ESM)

### Сложность: Средняя
### Производительность: Средняя-Высокая
### Рекомендуется для: Постепенного улучшения текущего пайплайна

---

## Вариант 6: PBR Forward Rendering

**Описание:** Forward rendering с Physically Based Rendering материалами.

### Архитектура:
1. **Main Pass** - Рендеринг с PBR шейдерами
2. **IBL Pass** - Image-Based Lighting (опционально)
3. **Post-Processing** - Tone mapping, bloom

### Преимущества:
- ✅ Реалистичное освещение
- ✅ Современный стандарт индустрии
- ✅ Хорошее качество визуализации

### Недостатки:
- ⚠️ Требует больше текстур (albedo, normal, metallic, roughness, AO)
- ⚠️ Более сложные шейдеры

### Реализация:
- PBR материалы (Metallic-Roughness workflow)
- IBL (Image-Based Lighting) с environment maps
- Tone mapping (ACES, Reinhard, etc.)

### Сложность: Средняя
### Производительность: Средняя
### Рекомендуется для: Проектов, требующих высокое качество визуализации

## Рекомендации по выбору

### Для текущего проекта (RenderRites Machine):

**Краткосрочная перспектива (1-2 месяца):**
1. **Вариант 5: Forward + Shadow Mapping** - Минимальные изменения, значительное улучшение качества
2. **Вариант 6: PBR Forward** - Современные материалы, улучшение визуального качества

**Среднесрочная перспектива (3-6 месяцев):**
1. **Вариант 1: Forward+** - Если требуется поддержка множества источников света
2. **Вариант 2: Deferred Rendering** - Если планируется сложная сцена с множеством источников света и PBR

**Долгосрочная перспектива (6+ месяцев):**
1. **Вариант 3: Hybrid Deferred-Forward** - Профессиональный пайплайн для сложных проектов

### Критерии выбора:

- **Количество источников света:**
  - < 20: Forward + Shadows
  - 20-100: Forward+
  - 100+: Deferred или Clustered

- **Требования к качеству:**
  - Базовое: Forward + Shadows
  - Высокое: PBR Forward
  - Профессиональное: Deferred + PBR

- **Ограничения по памяти:**
  - Низкие: Forward, Forward+
  - Средние: Clustered
  - Высокие: Deferred

## Дополнительные техники для любого пайплайна

### Post-Processing эффекты:
- **Bloom** - Свечение ярких объектов
- **Tone Mapping** - Корректировка экспозиции (ACES, Reinhard)
- **Color Grading** - Корректировка цветов
- **FXAA/TAA** - Сглаживание (Anti-aliasing)
- **SSAO** - Ambient Occlusion (для Deferred)
- **SSGI** - Global Illumination (для Deferred)
- **Motion Blur** - Размытие движения
- **Depth of Field** - Глубина резкости

### Оптимизации:
- **Occlusion Culling** - Отсечение невидимых объектов
- **Level of Detail (LOD)** - Уровни детализации
- **Texture Streaming** - Потоковая загрузка текстур
- **Shader Variants** - Варианты шейдеров для разных платформ

## План миграции (пример для Forward+)

### Этап 1: Подготовка (1 неделя)
- Изучение compute shaders в OpenGL
- Создание структуры данных для источников света
- Подготовка UBO для передачи данных

### Этап 2: Light Culling (2 недели)
- Реализация compute shader для разбиения на тайлы
- Реализация light culling логики
- Тестирование и отладка

### Этап 3: Интеграция (1 неделя)
- Интеграция в MainRenderSystem
- Обновление шейдеров для работы с тайлами
- Оптимизация производительности

### Этап 4: Тестирование и оптимизация (1 неделя)
- Профилирование
- Оптимизация
- Документация

**Общее время: ~5 недель**

---

## Заключение

Для RenderRites Machine рекомендуется начать с **Варианта 1 (Forward+)** или **Варианта 5 (Forward + Shadow Mapping)** в зависимости от приоритетов:

- Если нужны реалистичные тени → **Вариант 5**
- Если нужно много источников света → **Вариант 1**

Оба варианта можно комбинировать для получения оптимального результата.
